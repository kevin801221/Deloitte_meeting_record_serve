使用手冊

  1. 設置 OpenAI API 密鑰：點擊右上角的「設置 API 密鑰」按鈕，輸入您的 OpenAI API 密鑰。如果您沒有 API 密鑰，可以在 OpenAI 官網 申請。
  2. 檢查 API 狀態：頁面頂部顯示了三個 API 的狀態指示器，確保它們都是綠色的，表示 API 可用。

  1. 點擊「音頻轉文字」標籤。
  2. 上傳您的會議錄音文件（支持 WAV、MP3、M4A 等格式）。
  3. 輸入會議標題和參與者信息（可選）。
  4. 點擊「轉換為文字」按鈕。
  5. 等待系統處理，處理時間取決於音頻長度。
  6. 處理完成後，您將看到轉錄的文字結果。

  1. 點擊「音頻轉摘要」標籤。
  2. 上傳您的會議錄音文件。
  3. 輸入會議標題和參與者信息（可選）。
  4. 點擊「生成摘要」按鈕。
  5. 等待系統處理，這可能需要幾分鐘時間。
  6. 處理完成後，您將看到結構化的會議摘要，包括主要議題、決策和行動項目。

  1. 點擊「本地錄音」標籤。
  2. 點擊「開始錄音」按鈕，授予瀏覽器麥克風訪問權限。
  3. 開始您的會議或談話。
  4. 完成後，點擊「停止錄音」按鈕。
  5. 您可以點擊「播放」按鈕預覽錄音，或點擊「下載」按鈕保存錄音文件。
  6. 如果您想直接處理錄音，可以點擊「轉換為文字」或「生成摘要」按鈕。

  1. 點擊「文字轉摘要」標籤。
  2. 輸入或粘貼會議文字記錄。
  3. 輸入會議標題和參與者信息（可選）。
  4. 點擊「生成摘要」按鈕。
  5. 等待系統處理，這可能需要幾分鐘時間，取決於文字長度。
  6. 處理完成後，您將看到結構化的會議摘要。

Q: API 狀態顯示為紅色
A: 請確保您已設置有效的 OpenAI API 密鑰，並且 API 狀態指示器顯示為綠色。
Q: 上傳音頻文件時出現錯誤
A: 請確保您的音頻文件格式受支持（WAV、MP3、M4A 等），且文件大小不超過 100MB。
Q: 處理時間過長
A: 處理時間取決於音頻長度和文字數量。較長的會議錄音可能需要更多時間處理。
Q: 錄音功能無法使用
A: 請確保您已授予瀏覽器麥克風訪問權限，並且您的設備有可用的麥克風。
Q: 生成的摘要質量不佳
A: 摘要質量取決於音頻轉錄的準確性和會議內容的結構。嘈雜的環境或不清晰的發言可能影響結果。
設置 OpenAI API 密鑰
您可以使用系統提供的默認 API 密鑰，或設置您自己的 OpenAI API 密鑰。
您的 API 密鑰將安全地存儲在您的瀏覽器中，不會發送到我們的服務器。
